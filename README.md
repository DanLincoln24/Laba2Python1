## Обзор проекта

### Предметная область

Разработка гибкой подсистемы приёма задач в платформе обработки данных. Задачи могут поступать из различных источников, не связанных между собой наследованием, но обязанных реализовывать единый поведенческий контракт. 

**Ключевые особенности:**
- Duck typing и контрактное программирование через `typing.Protocol`
- Отсутствие общего базового класса для источников
- Runtime-проверка соблюдения контракта через `isinstance()`
- Полная аннотация типов для всех публичных интерфейсов
- Расширяемая архитектура (добавление новых источников без изменения существующего кода)

### Модель данных

**Task (Задача)**
- `id` — уникальный идентификатор (int или str)
- `payload` — произвольные данные задачи (Any)

## Архитектура и контракты

### Контракт источника задач (Protocol)

```python
@runtime_checkable
class TaskSource(Protocol):
    
    def get_tasks(self) -> Iterable[Task]:
```
## Описание компонентов
````
Task --- Структура данных для хранения задачи
TaskSource (Protocol) --- Контракт, который должны соблюдать все источники
FileTaskSource --- Читает задачи из JSON-файла
RandomTaskSource --- Генерирует случайные задачи
ApiStubSource --- Имитирует получение задач от API
process_tasks --- Функция обработки задач из источника
safe_process --- Функция с проверкой соблюдения контракта
````
## Реализация источников
Все источники реализованы как независимые классы без общего предка.

### FileTaskSource (Файловый источник)
Читает задачи из JSON-файла. 
#### Методы:

__init__(filename: str) — сохраняет путь к файлу

get_tasks() -> Iterable[Task] — открывает файл, читает JSON и возвращает задачи

RandomTaskSource (Генератор случайных задач)
Создаёт заданное количество задач со случайными payload'ами:

ID — порядковый номер (0, 1, 2...)

### Payload — случайная строка из 5 букв

#### Методы:

__init__(count: int) — задаёт количество задач

get_tasks() -> Iterable[Task] — генерирует задачи

### ApiStubSource (Заглушка API)
Имитирует получение задач от внешнего API. Возвращает фиксированный набор задач:
ID: 'api_1', 'api_2'

Payload: словарь с ключами 'source' и 'data'

#### Методы:

__init__(base_url: str) — сохраняет URL (для имитации)

get_tasks() -> Iterable[Task] — возвращает фиксированные задачи
